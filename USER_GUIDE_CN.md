## 产品简介

YesnoteLite 是一款简洁高效的 Markdown 笔记应用，专为需要快速记录和组织想法的用户设计。

### 核心特性

- 🎨 **所见即所得编辑器** - 基于 EditorJS，支持富文本编辑
- 📁 **文件夹组织** - 灵活的文件夹结构，支持无限层级
- 🔄 **Git 同步** - 内置 GitHub 同步，自动版本控制
- ⏱️ **自动保存** - 每 30 秒自动保存，永不丢失数据
- 🔍 **全文搜索** - 快速查找所有笔记内容
- ⭐ **收藏与置顶** - 重要笔记一键收藏和置顶
- 📜 **版本历史** - 基于 Git 的完整版本历史记录
- ⌨️ **快捷键支持** - 高效的键盘操作

### 系统要求

- **macOS**: 10.12 或更高版本
- **磁盘空间**: 至少 200 MB
- **内存**: 建议 4 GB 或以上

## 快速开始

### 安装

1. 下载对应芯片的安装包：
   - Apple Silicon (M1/M2/M3): `YesnoteLite-0.1.0-arm64.dmg`
   - Intel 芯片: `YesnoteLite-0.1.0-x64.dmg`

2. 双击 DMG 文件，将应用拖到 Applications 文件夹

3. 首次打开时，如果遇到"无法打开"提示：
   - 打开 `系统偏好设置` → `安全性与隐私`
   - 点击"仍要打开"按钮
   - 或在终端运行: `xattr -cr /Applications/YesnoteLite.app`

### 首次使用

1. 启动应用后，会自动在 `~/YesnoteLite/repos/default/` 创建默认笔记本
2. 左侧是文件树，显示所有笔记和文件夹
3. 右侧是编辑器，可以开始创建笔记
4. 点击左上角的 `+` 按钮创建根文件夹

## 核心功能

### 1. 笔记管理

#### 创建笔记

**方法一：通过文件树**
1. 在左侧文件树中找到目标文件夹
2. 点击文件夹旁的 ⚙️ 图标
3. 选择"创建文件"
4. 输入文件名，点击确定

**方法二：通过根文件夹**
1. 点击左侧底部的 ➕ 按钮
2. 创建新的根文件夹
3. 在新文件夹中创建笔记

#### 编辑笔记

1. 在左侧文件树中点击笔记名称
2. 笔记内容会在右侧编辑器中打开
3. 直接编辑标题和内容
4. 应用会每 30 秒自动保存

#### 删除笔记

1. 在左侧文件树中找到要删除的笔记
2. 点击笔记旁的 ⚙️ 图标
3. 选择"删除"
4. 确认删除操作

**注意**: 删除操作不可撤销，请谨慎操作！

### 2. 文件夹管理

#### 创建文件夹

**创建根文件夹**
1. 点击左侧底部的 ➕ 按钮
2. 输入文件夹名称
3. 点击确定

**创建子文件夹**
1. 点击父文件夹旁的 ⚙️ 图标
2. 选择"创建子文件夹"
3. 输入文件夹名称
4. 点击确定

#### 组织结构

- 支持无限层级的文件夹嵌套
- 建议不超过 3-4 层，保持结构清晰
- 根文件夹最多 6 个

**推荐的文件夹结构**
```
📁 工作
  📁 项目A
    📄 需求文档
    📄 会议记录
  📁 项目B
📁 学习
  📁 编程
  📁 设计
📁 个人
  📄 日记
  📄 想法
```

### 3. 编辑器功能

#### 添加内容块

1. 点击编辑器左侧的 `+` 按钮
2. 从菜单中选择块类型
3. 开始输入内容

#### 块类型详解

**段落（Paragraph）**
- 基本文本块
- 支持内联格式：粗体、斜体、下划线、链接

**标题（Header）**
- 6 个级别：H1 到 H6
- 快捷键: `Cmd+Shift+H`
- 用于文档结构化

**列表（List）**
- 有序列表（1, 2, 3...）
- 无序列表（• • •）
- 支持多层嵌套

**待办清单（Checklist）**
- ☐ 未完成项
- ☑ 已完成项
- 快捷键: `Cmd+Shift+K`
- 适合任务管理

**引用（Quote）**
- 用于引用他人的话
- 或强调重要内容

**代码块（Code）**
- 用于代码片段
- 保留格式和缩进
- 快捷键: `Cmd+Shift+C`

**表格（Table）**
- 创建结构化数据
- 可调整行列数
- 快捷键: `Cmd+Shift+T`
- 支持表头

**警告框（Warning）**
- 标题 + 消息
- 用于重要提示
- 快捷键: `Cmd+Shift+W`

#### 文本格式化

选中文本后会出现内联工具栏：

- **粗体**: 选中文本，点击 **B** 或按 `Cmd+B`
- **斜体**: 选中文本，点击 *I* 或按 `Cmd+I`
- **下划线**: 选中文本，点击 <u>U</u>
- **链接**: 选中文本，点击 🔗，输入 URL

#### 块操作

- **移动块**: 拖拽块左侧的拖拽手柄
- **删除块**: 点击块右侧的 ⚙️，选择删除
- **转换块**: 点击块右侧的 ⚙️，选择转换为其他类型

### 4. 搜索功能

#### 全文搜索

1. 点击顶部搜索栏或按 `Cmd+F`
2. 输入搜索关键词
3. 实时显示搜索结果
4. 点击结果打开对应笔记

#### 搜索技巧

- 搜索会匹配笔记标题和内容
- 支持中英文搜索
- 不区分大小写
- 按 ESC 清空搜索并关闭结果

#### 过滤文件树

1. 在左侧文件树上方的过滤框中输入
2. 文件树会只显示匹配的笔记
3. 清空输入框恢复完整显示

### 5. 收藏功能

#### 置顶笔记

**置顶操作**
1. 右键点击笔记或点击 ⚙️ 图标
2. 选择"置顶笔记"
3. 笔记会移到文件夹顶部

**取消置顶**
1. 右键点击已置顶的笔记
2. 选择"取消置顶"

**用途**
- 快速访问正在进行的项目
- 突出显示重要文档
- 临时提高某些笔记的优先级

#### 收藏笔记

**收藏操作**
1. 右键点击笔记或点击 ⚙️ 图标
2. 选择"收藏笔记"
3. 笔记旁会显示 ⭐ 图标

**查看收藏**
1. 点击左侧底部的 ⭐ 按钮
2. 收藏侧边栏会滑出
3. 显示所有收藏的笔记
4. 点击笔记快速打开

**取消收藏**
1. 右键点击已收藏的笔记
2. 选择"取消收藏"

**区别**
- **置顶**: 在当前文件夹内排序靠前
- **收藏**: 全局收藏夹，跨文件夹访问

## Git 同步

### 设置 GitHub 同步

#### 准备工作

1. **创建 GitHub 仓库**
   - 登录 GitHub.com
   - 点击右上角 `+` → `New repository`
   - 输入仓库名（如 `my-notes`）
   - 选择 Private（私有）
   - 不要初始化 README
   - 点击 `Create repository`

2. **创建 Personal Access Token**
   - 进入 GitHub Settings → Developer settings
   - 选择 Personal access tokens → Tokens (classic)
   - 点击 `Generate new token (classic)`
   - 勾选权限：`repo` (完整仓库访问权限)
   - 点击 `Generate token`
   - **复制 token**（只显示一次，请妥善保存）

#### 配置应用

1. 点击左侧底部的 ⚙️ 设置按钮
2. 切换到"远程设置"标签
3. 填写以下信息：
   - **GitHub 用户名**: 你的 GitHub 用户名
   - **GitHub Token**: 刚才创建的 token
   - **仓库名称**: 你创建的仓库名（如 `my-notes`）
4. 点击"从 GitHub 初始化"
5. 等待克隆完成

### 使用 Git 同步

#### 推送更改

**方法一：使用快捷键**
- 按 `Cmd+Shift+P`
- 应用会自动提交并推送所有更改

**方法二：使用 Git 按钮**
1. 点击左侧底部的 🔄 Git 按钮
2. 选择"推送"
3. 等待推送完成

**方法三：使用状态指示器**
1. 顶部会显示未推送的更改数量
2. 点击状态指示器
3. 自动推送

#### 拉取更新

1. 点击左侧底部的 🔄 Git 按钮
2. 选择"拉取"
3. 等待拉取完成
4. 文件树会自动刷新

#### Git 状态说明

**顶部状态指示器**
- 🟡 黄色 + 数字: 有未推送的更改
- 🔵 蓝色: 正在检查状态
- ⚪ 灰色: 没有更改

**底部状态栏**
- "推送中...": 正在推送到 GitHub
- "推送成功": 推送完成
- "拉取中...": 正在从 GitHub 拉取
- "拉取成功": 拉取完成

### 自动同步

- 切换到远程笔记本时，自动拉取最新更改
- 保存笔记后，Git 状态会自动更新
- 每次推送都会自动提交所有更改

### 多设备同步

1. 在设备 A 上编辑笔记并推送
2. 在设备 B 上打开应用
3. 切换到远程笔记本（自动拉取）
4. 或手动点击"拉取"按钮
5. 获取最新的笔记内容

### 冲突处理

如果在多设备上同时编辑同一笔记：

1. 应用会提示有冲突
2. 建议：
   - 在一个设备上完成编辑后推送
   - 在另一个设备上先拉取再编辑
3. 如遇到 Git 冲突，可能需要手动解决

## 版本历史

### 查看版本历史

1. 打开一个笔记
2. 点击顶部工具栏的 🕐 历史按钮
3. 右侧会滑出版本历史面板

### 历史记录信息

每条历史记录显示：
- 📅 提交时间
- 👤 提交者（通常是你的 GitHub 用户名）
- 💬 提交信息（自动生成）
- #️⃣ 提交哈希值（前 7 位）

### 预览历史版本

1. 在历史记录中点击"预览"按钮
2. 弹出对话框显示该版本的内容
3. 以只读模式显示
4. 可以查看当时的笔记内容

### 恢复历史版本

1. 在历史记录中点击"恢复"按钮
2. 确认恢复操作
3. 当前笔记内容会被替换为历史版本
4. 编辑器会自动更新显示

**注意**:
- 恢复操作会覆盖当前内容
- 建议先预览确认再恢复
- 恢复后会创建新的提交记录

### 版本历史限制

- 只显示最近 50 次提交
- 只有远程笔记本（Git 仓库）才有版本历史
- 本地笔记本不支持版本历史

## 高级功能

### 笔记本管理

#### 创建新笔记本

1. 点击左侧底部的 ⚙️ 设置按钮
2. 在"全局设置"标签下
3. 点击"初始化默认笔记本"
4. 会在默认路径创建新笔记本

#### 切换笔记本

1. 打开设置
2. 在"当前笔记本"下拉菜单中选择
3. 选择要切换的笔记本
4. 点击"确定"保存

#### 笔记本类型

**本地笔记本**
- 存储在本地磁盘
- 不支持 Git 同步
- 不支持版本历史
- 适合私密内容

**远程笔记本**
- 连接到 GitHub 仓库
- 支持 Git 同步
- 支持版本历史
- 适合多设备同步

### 更改存储路径

1. 打开设置
2. 在"本地路径"栏点击"更改默认路径"
3. 选择新的文件夹
4. 应用会将笔记移动到新位置

### 编辑原始 JSON

**⚠️ 高级功能，谨慎使用**

1. 点击左上角的 📄 按钮
2. 打开原始 JSON 编辑器
3. 可以直接编辑笔记的 JSON 数据
4. 保存后更新编辑器显示

**用途**:
- 批量修改数据
- 调试编辑器问题
- 导入/导出特殊格式

**风险**:
- 错误的 JSON 格式会导致笔记无法打开
- 建议先备份笔记

### 开发者工具

1. 点击左上角的 🖥️ 按钮
2. 打开 Chrome DevTools
3. 可以查看控制台日志
4. 调试应用问题

## 常见问题

### 安装与启动

**Q: 双击应用提示"已损坏，无法打开"？**

A: 这是 macOS 的安全机制。解决方法：
```bash
xattr -cr /Applications/YesnoteLite.app
```
然后重新打开应用。

**Q: 应用无法启动？**

A: 检查：
1. macOS 版本是否 ≥ 10.12
2. 是否下载了正确的芯片版本（Intel/Apple Silicon）
3. 尝试重新下载安装包

### 笔记管理

**Q: 笔记保存在哪里？**

A: 默认路径是 `~/YesnoteLite/repos/default/`
可以在 Finder 中按 `Cmd+Shift+G` 输入路径访问。

**Q: 如何备份笔记？**

A: 三种方法：
1. 复制整个 `~/YesnoteLite` 文件夹
2. 使用 Git 同步到 GitHub
3. 使用 Time Machine 备份

**Q: 可以导入 Markdown 文件吗？**

A: 当前版本不支持直接导入。建议：
1. 手动复制粘贴内容
2. 或将 .md 文件放到笔记本文件夹中

**Q: 删除的笔记可以恢复吗？**

A:
- 如果使用 Git 同步：可以通过版本历史恢复
- 如果是本地笔记本：无法恢复，请谨慎删除

### Git 同步

**Q: 推送失败怎么办？**

A: 检查：
1. 网络连接是否正常
2. GitHub Token 是否有效
3. 仓库权限是否正确
4. 查看底部状态栏的错误信息

**Q: 如何查看 GitHub Token？**

A: Token 只在创建时显示一次。如果忘记：
1. 在 GitHub 删除旧 Token
2. 创建新的 Token
3. 在应用中重新配置

**Q: 可以同步到 GitLab 或其他平台吗？**

A: 当前版本只支持 GitHub。

**Q: 同步会消耗多少流量？**

A: 笔记是纯文本，流量很小：
- 一篇 1000 字的笔记约 5-10 KB
- 图片会占用更多流量

### 编辑器

**Q: 工具栏的 + 按钮不显示？**

A: 这通常是因为：
1. 编辑器正在加载，稍等片刻
2. 尝试刷新应用（`Cmd+R`）
3. 重启应用

**Q: 如何插入图片？**

A: 当前版本暂不支持图片插入。计划在未来版本中添加。

**Q: 支持 Markdown 语法吗？**

A: 部分支持：
- 使用 `#` 开头会自动转换为标题
- 使用 `-` 或 `*` 开头会自动转换为列表
- 其他 Markdown 语法需要手动选择块类型

**Q: 如何导出为 PDF 或其他格式？**

A: 当前版本不支持导出。建议：
1. 复制内容到其他应用（如 Pages）
2. 使用其他工具导出
3. 或直接使用 GitHub 上的 Markdown 文件

### 性能

**Q: 应用运行缓慢？**

A: 优化建议：
1. 减少单个笔记本中的笔记数量（建议 < 1000 个）
2. 定期清理不需要的笔记
3. 关闭不使用的功能（如自动保存）
4. 重启应用

**Q: 搜索很慢？**

A:
1. 搜索会扫描所有笔记内容
2. 笔记越多，搜索越慢
3. 建议使用文件树过滤功能

## 技巧与最佳实践

### 组织笔记

#### 推荐的文件夹结构

```
📁 工作
  📁 项目A
    📁 需求
    📁 设计
    📁 开发
  📁 项目B
  📁 会议记录

📁 学习
  📁 编程
    📁 JavaScript
    📁 Python
  📁 设计
  📁 读书笔记

📁 个人
  📁 日记
  📁 想法
  📁 待办事项
```

#### 命名规范

**文件夹命名**
- 使用简短、描述性的名称
- 避免特殊字符
- 可以使用中文或英文

**笔记命名**
- 格式：`日期-主题`，如 `2024-02-14-项目需求`
- 或：`主题-版本`，如 `API文档-v2`
- 保持一致的命名风格

### 高效工作流

#### 快速记录

1. 使用置顶功能固定"快速记录"笔记
2. 随时打开记录想法
3. 定期整理到对应文件夹

#### 项目管理

1. 为每个项目创建独立文件夹
2. 使用待办清单跟踪任务
3. 会议记录用日期命名
4. 重要文档添加收藏

#### 知识管理

1. 按主题分类（不要太细）
2. 使用标题组织笔记结构
3. 相关笔记之间添加链接
4. 定期回顾和更新

### 备份策略

#### 三重备份

1. **本地备份**: 定期复制 YesnoteLite 文件夹
2. **云端备份**: 使用 GitHub 同步
3. **时间机器**: 使用 macOS Time Machine

#### 备份频率

- 重要项目：每天备份
- 一般笔记：每周备份
- 归档内容：每月备份

### 多设备协作

#### 同步流程

**设备 A（主要设备）**
1. 编辑笔记
2. 保存（自动）
3. 推送到 GitHub（`Cmd+Shift+P`）

**设备 B（辅助设备）**
1. 打开应用
2. 拉取更新（自动或手动）
3. 查看/编辑笔记
4. 推送更改

#### 避免冲突

- 在一个设备上完成编辑后再切换
- 每次切换设备前先推送
- 每次开始工作前先拉取

### 搜索技巧

#### 有效搜索

- 使用关键词而非完整句子
- 尝试不同的关键词组合
- 利用文件树过滤缩小范围

#### 标签系统

虽然没有内置标签功能，可以使用：
- 在笔记标题中添加标签：`[项目] 需求文档`
- 在笔记开头添加标签行：`#项目 #需求`
- 通过搜索标签快速定位

### 提高效率

#### 熟练使用快捷键

- 记住常用快捷键（`Cmd+S`, `Cmd+Shift+P`, `Cmd+?`）
- 减少鼠标使用
- 提高编辑速度

#### 自定义工作流

1. 创建笔记模板（在固定笔记中）
2. 复制模板创建新笔记
3. 保持统一的笔记格式

#### 定期维护

- 每周清理不需要的笔记
- 整理文件夹结构
- 更新重要笔记
- 检查 Git 同步状态

## 附录

### 文件格式

笔记以 JSON 格式存储：

```json
{
  "time": 1676390400000,
  "blocks": [
    {
      "type": "header",
      "data": {
        "text": "标题",
        "level": 1
      }
    },
    {
      "type": "paragraph",
      "data": {
        "text": "段落内容"
      }
    }
  ],
  "version": "2.26.5"
}
```

### 系统文件

- **配置文件**: `~/Library/Application Support/YesnoteLite/config.json`
- **笔记文件**: `~/YesnoteLite/repos/[笔记本名称]/`
- **日志文件**: `~/Library/Logs/YesnoteLite/`

### 隐私与安全

- 所有数据存储在本地
- Git 同步使用 HTTPS 加密
- GitHub Token 存储在本地配置文件中
- 建议使用私有仓库存储笔记
